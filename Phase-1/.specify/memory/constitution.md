# In-memory Todo Console Application (Phase I) Constitution

## Core Principles

### I. Specification-First Development
All functionality must originate from written specifications.

### II. Agentic Code Generation
All code MUST be generated by the specified agentic framework (e.g., Gemini Code). No manual coding is permitted.

### III. Deterministic, Reviewable Workflows
All development activities (spec writing, code generation, testing) MUST be managed via the Spec-Kit Plus framework, with complete history preserved in the repository.

### IV. Clean, Modular Design
The application MUST adhere to clean, modular Python design principles.

### V. Robust User Experience
The console interface MUST be clear, simple, and provide explicit error handling for all invalid user inputs.

### VI. Technology Constraints
The project is strictly limited to Python 3.13+, with an in-memory data model (no persistence), a console-based interface, and no external frameworks.

### VII. Scope Limitation
The application scope is strictly limited to basic CRUD operations (add, view, update, delete) and managing completion status for tasks.

## Governance
This Constitution supersedes all other practices. Amendments require documentation, formal approval, and a clear migration plan. All pull requests and code reviews must verify compliance with these principles.
